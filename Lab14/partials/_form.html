Preguntas a responder

¿Por qué es importante hacer un session_unset() y luego un session_destroy()?

Es buena práctica primero limpiar todas las variables de la sesión antes de eliminarla.

¿Cuál es la diferencia entre una variable de sesión y una cookie?

Las variables de session guardan los datos en la memoria del servidor web, Las cookies guardan la info en un archivo en el cliente y solo pueden conservar un tamaño limitado de informacion, en cambio la session puede conservar objetos complejos

¿Qué técnicas se utilizan en sitios como facebook para que el usuario no sobreescriba sus fotos en el sistema de archivos cuando sube una foto con el mismo nombre?

Simplemente es necesario agregar un 'timestamp' al final del nombre del archivo para que de esa forma todos los archivos subidos tengan un nombre distinto pues jamás se podrán subir dos archivos con el mismo 'timestamp'.

¿Qué es CSRF y cómo puede prevenirse?

Es un tipo de exploit malicioso de un sitio web en el que comandos no autorizados son transmitidos por un usuario en el cual el sitio web confía. Puede prevenirse con diferentes métodos como: Doble-posted cookie (envío doble de cookie), Unique form nonce (formulario único para cada solicitud), Requerir credenciales de autentificación nuevamente, One time Passwords.
